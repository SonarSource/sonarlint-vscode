### Rules



1. Javascript: Files should not have too many lines of code

2. Javascript: Functions should not have too many parameters

3. Javascript: Magic numbers should not be used

4. Javascript: String literals should not be duplicated

5. Javascript: Functions should not have too many lines of code

6. Javascript: Loop counters should not be assigned within the loop body

7. Javascript: "Array.prototype.sort()" and "Array.prototype.toSorted()" should use a compare function

8. Javascript: Array-mutating methods should not be used misleadingly

9. Javascript: The "any" type should not be used

10. Javascript: Regular expressions should not contain control characters

11. Javascript: Unnecessary character escapes should be removed

12. Javascript: Ends of strings should be checked with "startsWith()" and "endsWith()"

13. Javascript: Optional chaining should be preferred

14. Javascript: If statements should not be the only statement in else blocks

15. Javascript: Literals should not be used for promise rejection

16. Javascript: Mutable variables should not be exported

17. Javascript: Functions should be moved to the highest possible scope

18. Javascript: Promise.resolve() and Promise.reject() should not be used in async functions or promise callbacks

19. Javascript: DOM nodes should be removed using "remove()" instead of "removeChild()"

20. Javascript: Top-level await should be preferred over wrapped async operations
